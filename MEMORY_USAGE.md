# Memory Usage - Tang Nano 4K (GW1NSR-4C)

This document defines the formal memory usage plan for the various RAM and Flash regions available on the Tang Nano 4K when running MicroPython.

## 1. Internal SRAM (22 KB)

- **Region**: `0x20000000 - 0x200057FF`
- **Role**: Primary Heap, Stack, and Static Data.
- **MicroPython Usage**:
    - **Stack**: 2 KB (at the top of SRAM: `0x20005000 - 0x200057FF`).
    - **Data/BSS**: ~1-2 KB (statically allocated data).
    - **Fast Heap**: ~18 KB (initialized via `gc_init()`). This is the most efficient memory for object allocation.
- **Performance**: Highest (zero wait states, connected via AHB-Lite).

## 2. External PSRAM (8 MB)

- **Region**: `0xA0000000 - 0xA07FFFFF`
- **Role**: Large/Extended Heap.
- **MicroPython Usage**:
    - **Large Heap**: Initialized via `gc_add()`.
    - Provides ~8 MB of additional space for large Python objects, framebuffers, and complex data structures.
- **Implementation**: The **Gowin PSRAM Memory Interface** IP must be instantiated in the FPGA bitstream and configured for **Memory Mapped Mode** (AHB interface).
- **Performance**: Slower than internal SRAM due to bus latency and FPGA-to-PSRAM timing.

## 3. Flash Memory

The Tang Nano 4K utilizes both internal SoC flash and external SPI Flash.

### 3.1 Internal Flash (32 KB / 128 KB)

- **Hardware Limit**: The GW1NSR-LV4C SoC has **32 KB** of physical internal instruction flash (ITCM).
- **Address Space**: The SoC provides a **128 KB** address space at `0x00000000 - 0x0001FFFF`.
- **Standard Usage**: In non-SPLIT_FLASH builds, the entire firmware must fit within the 128KB limit (simulated) or 32KB limit (hardware).
- **SPLIT_FLASH Usage**: Used for the Interrupt Vector Table (IVT), Reset Handler, and low-level initialization.

### 3.2 External SPI Flash (4 MB)

- **Region**: `0x60000000` (Memory Mapped via FPGA IP).
- **Capacity**: 4 MB (W25Q64 or similar).
- **Role**: MicroPython Runtime (Code/ROData) and Filesystem.

#### SPLIT_FLASH Architecture
To overcome the 32 KB internal flash limit, the `SPLIT_FLASH` mode is used:
- **Internal Flash (0x00000000)**: First 32 KB for bootloader and vectors.
- **External Flash (0x60000000)**: The main MicroPython `.text` and `.rodata` sections are linked here.

#### LittleFS Filesystem
The external flash also hosts the LittleFS2 filesystem:
- **Base Address**: `0x60000000`.
- **FS Offset**:
    - **Standard**: `0x100000` (1 MB offset) to avoid overlapping with small firmware.
    - **SPLIT_FLASH**: `0x40000` (256 KB offset) to accommodate larger firmware residing in external flash.
- **FS Size**: Remainder of the 4 MB flash (~3 MB or ~3.75 MB).

## 4. Summary Table

| Region | Capacity | Base Address | Role | MicroPython Usage |
| :--- | :--- | :--- | :--- | :--- |
| **Internal SRAM** | 22 KB | `0x20000000` | Fast RAM | Stack, Static Data, Fast Heap |
| **External PSRAM** | 8 MB | `0xA0000000` | Large RAM | Extended Heap |
| **Internal Flash** | 32 KB* | `0x00000000` | Boot Flash | Vectors, Bootloader, Init |
| **External Flash** | 4 MB | `0x60000000` | Main Storage | Runtime Code, ROData, VFS (LittleFS) |

*\* Note: Internal Flash address space is 128 KB, but physical hardware is limited to 32 KB.*

## 5. Implementation Notes

- **Bus Access**: Memory-mapped access to PSRAM (`0xA0000000`) and External Flash (`0x60000000`) requires specific IP cores (Gowin PSRAM and SPI Flash Interface) to be present in the FPGA bitstream.
- **Linker Configuration**: Controlled via `tang_nano_4k.ld` and `regions.ld` (generated by the Makefile).
- **Heap Management**: MicroPython is configured with `MICROPY_GC_SPLIT_HEAP=1` to manage the two distinct RAM regions.
